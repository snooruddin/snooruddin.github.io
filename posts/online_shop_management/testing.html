Testing The Insertions

Testing the insertions in pretty easy. If you are using TOAD, you can use the run script button on the navigation bar. But You can also run a file in the using the "Run SQL Command Line" application. To run the file in the command line:

  1. Login to your user account in the command line app

  2. Enter the character '@' into the command line and then drag and drop your file to the command window.

                                    
                                    @ 'your_file_path_here';

    If you press enter then, the entire file will be run.

Testing the functions and procedures

There are two kinds of blocks in PL/SQL:

  1. Named Block : As the name suggests, named blocks have names. As Oracle uses a client-server architecture, the named blocks are sent to the server. The server tests the named block for errors, saves the block and assigns a pointer to the memory location of the block statements to the name of the block. The server checks the named block only the first time the block is sent unless you make changes to the internal statements of the named block later. If you don't change the internal statements of your named block later, when you request your client for running the named blocks by envoking their names, the server only runs the statements on the memory location assigned to that name.

  2. Unnamed Block: Unnamed blocks are sent everytime to the server as you invoke them. The server checks them each time for errors and then runs them.

We normally test the procedures and functions using unnamed blocks. The structure of an unnamed block is as follows : 

                        
                            DECLARE
                                --declare the variables you want to use in this unnamed block
                            BEGIN
                                --write your statements here
                            END
Testing a function : getUnderstockedProducts 
                        
                            SET SERVEROUTPUT ON;
                            DECLARE
                                totalNum NUMBER;
                            BEGIN
                                --calling the function
                                totalNum := getUnderstockedProducts;
                                DBMS_OUTPUT.PUT_LINE('Total Understocked Product : ' || totalNum);
                            END;
Testing a Procedure : awardClubPointsToAllCustomers 
                        
                            SET SERVEROUTPUT ON;
                            DECLARE
                                awardAmnt NUMBER;
                                ttlAwarded INTEGER;
                            BEGIN
                                --calling the procedure
                                --notice how we are providing the output parameter too
                                awardClubPointsToAllCustomers(awardAmnt, ttlAwarded);

                                --the output parameter passed to the procedure will contain the data intended to be the output
                                dbms_output.PUT_LINE('Total Awarded To : ' || ttlAwarded);
                            END;

Testing Triggers

Testing triggers are easy. After you successfully execute the trigger without any errors or warnings, you insert data to the table related to that trigger. Then check whether the trigger has done what it was supposed to do.
